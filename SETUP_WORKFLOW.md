# Daily Repository Status Report Workflow - Setup Guide

## What was created

A GitHub Agentic Workflow that generates a **daily repository status report** for maintainers every weekday. The workflow analyzes repository activity and creates a comprehensive issue summarizing:

- Pull request metrics and status
- Issue summary and triage needs  
- Recent commits and contributor activity
- Actionable recommendations for maintainers

## Files created

1. **`.github/workflows/daily-repo-status.md`** - The workflow definition (markdown + YAML frontmatter)
   - Describes what the AI agent should analyze and report
   - Contains all configuration and instructions
   - Can be edited directly without recompilation

2. **`.github/workflows/daily-repo-status.lock.yml`** - Compiled GitHub Actions workflow
   - Auto-generated by `gh aw compile`
   - Currently contains setup instructions
   - Will be generated with actual workflow logic once gh-aw is installed

3. **`.gitattributes`** - Git configuration for lock files
   - Marks lock files as generated (prevents merge conflicts)
   - Required for gh-aw workflows

## Next steps to activate

### 1. Install GitHub Agentic Workflows CLI

```bash
curl -sL https://raw.githubusercontent.com/github/gh-aw/main/install-gh-aw.sh | bash
```

Verify installation:
```bash
gh aw version
```

### 2. Compile the workflow

```bash
cd /Users/Aleksandr_Kirilushkin/Desktop/hello-world
gh aw compile daily-repo-status --strict
```

This generates the actual `.github/workflows/daily-repo-status.lock.yml` file.

### 3. Commit and push

```bash
git add .gitattributes .github/workflows/daily-repo-status.md .github/workflows/daily-repo-status.lock.yml
git commit -m "Initialize daily repository status report workflow"
git push
```

### 4. Trigger manually or wait for schedule

Once pushed to GitHub, the workflow will:
- Run automatically every weekday (fuzzy-scheduled time to distribute load)
- Can be triggered manually anytime via `workflow_dispatch`
- Create an issue with the daily status report each run

## What the workflow does

When the workflow runs, the AI agent:

1. **Queries repository state** using GitHub MCP tools:
   - Open/merged pull requests (including review status and CI status)
   - Open/closed issues 
   - Recent commits and contributors
   - Build/CI results

2. **Analyzes activity** over the last 24 hours:
   - Detects stale PRs awaiting review
   - Identifies failing CI/CD pipelines
   - Tracks newly filed issues needing triage
   - Measures contributor velocity

3. **Generates a report** with:
   - Summary statistics
   - Actionable insights for maintainers
   - Highlighted PRs/issues requiring attention
   - Trend analysis (comparing to previous days via cache-memory)

4. **Creates an issue** with the report, titled `[Daily Status] Repository Report - YYYY-MM-DD`

## Customization

The workflow markdown body (everything after the `---` frontmatter) is a **prompt that can be edited anytime**:

- Change what metrics to track
- Modify the report format
- Add custom analysis logic
- Adjust tone and detail level

**No recompilation needed** — changes take effect on the next run.

To change **configuration** (triggers, permissions, tools, timeouts):
- Edit the YAML frontmatter section (between `---` markers)
- Recompile: `gh aw compile daily-repo-status --strict`
- Commit the updated `.lock.yml`

## Key features

✅ **Scheduled daily** on weekdays to avoid Monday backlog  
✅ **Comprehensive metrics** - PRs, issues, commits, CI status  
✅ **Human attribution** - Credits team members driving the activity  
✅ **Trends over time** - Uses cache-memory to detect patterns  
✅ **Maintainer-focused** - Highlights actionable items  
✅ **Easy to customize** - Edit prompt body without recompilation
